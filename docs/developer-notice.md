# MCP Gateway 개발자 공지

**대상**: MCP Gateway를 통해 AI 코딩 도구(opencode)를 사용하는 개발자  
**배포일**: 2026-03-31  
**분류**: 보안 공지

---

## 📢 MCP Gateway 운영 안내

안녕하세요.

개발팀의 AI 코딩 도구(opencode) 환경에 **MCP Gateway**가 적용되었습니다.  
MCP Gateway는 여러분이 사용하는 Context7, Exa 검색 서비스를 보안 정책에 맞게 안전하게 이용할 수 있도록 지원하는 중계 시스템입니다.

---

## ✅ 무엇이 달라지나요?

### 이전

```
opencode → context7.com / api.exa.ai (직접 연결)
```

### 이후

```
opencode → MCP Gateway → context7.com / api.exa.ai
```

**개발자 여러분이 느끼는 차이는 없습니다.**  
기존과 동일하게 opencode에서 질문하면 자동으로 검색이 이루어집니다.  
단, 아래의 보안 정책이 자동으로 적용됩니다.

---

## 🛡️ 자동 차단 정책

검색 요청에 아래 유형의 정보가 포함되면, **외부 서비스로 전송되기 전에 자동으로 차단** 처리됩니다.

| 카테고리 | 차단 대상 예시 | 처리 방식 |
|----------|-----------------|-----------|
| 개인식별정보 | 주민등록번호 (`900101-1234567`) | 검색 차단 (403) |
| 개인연락처 | 휴대폰 번호 (`010-1234-5678`) | 검색 차단 (403) |
| 이메일 주소 | `user@company.com` | 검색 차단 (403) |
| 내부 네트워크 | 사설 IP (`10.x.x.x`, `192.168.x.x`, `172.16.x.x`) | 검색 차단 (403) |
| 내부 도메인 | `server.internal`, `db.corp` | 검색 차단 (403) |
| DB 접속정보 | `postgres://user:password@host` | 검색 차단 (403) |
| 인증 자격증명 | `API_KEY`, `SECRET_TOKEN`, `password=...` | 검색 차단 (403) |
| 클라우드 키 | AWS 액세스 키 (`AKIA...`) | 검색 차단 (403) |

### 차단 예시

**입력한 검색어:**
```
10.10.20.30 서버에서 Server not found 에러가 발생했어
```

**외부로 전송되는 검색어 (차단 시):**
```
차단된 서버에서 Server not found 에러가 발생했어
```

**결과:** 검색은 정상적으로 진행되고, 내부 IP는 외부에 노출되지 않습니다.

---

## ⚠️ 주의사항

### 1. 검색 결과가 부정확할 수 있습니다

차단으로 인해 검색어의 의미가 일부 변경될 수 있습니다. 예를 들어:
- 특정 서버 IP로 검색 → 차단되어 일반적인 결과 반환
- 이 경우, IP 대신 **서버 역할이나 에러 메시지**를 중심으로 검색하면 더 정확한 결과를 얻을 수 있습니다.

**비효율적인 검색:**
```
10.10.20.30에서 ORA-12541 에러 발생
```

**효율적인 검색:**
```
Oracle DB TNS Listener ORA-12541 에러 해결 방법
```

### 2. 검색 이력이 기록됩니다

모든 검색 요청은 보안 모니터링 목적으로 기록됩니다.

- 기록 항목: 요청 시간, 요청 IP, 서비스, 검색어 (원본), 차단 여부
- 목적: 보안 사고 대응 및 정책 개선
- 열람 권한: 보안 담당자

### 3. 차단은 보안 위반이 아닙니다

차단은 여러분이 실수로 민감 정보를 검색어에 포함시키더라도 외부로 유출되지 않도록 **자동으로 보호**하는 기능입니다. 차단이 발생했다고 해서 보안 위반으로 처리되지 않습니다.

## 🔧 개발자 PC 설정 방법

### opencode 설정 (`%APPDATA%\opencode\opencode.json`)

```json
{
  "mcp": {
    "context7": {
      "type": "local",
      "command": ["npx", "-y", "@upstash/context7-mcp"],
      "environment": {
        "CONTEXT7_API_URL": "http://{gateway-host}:8000/proxy/c7",
        "CONTEXT7_API_KEY": "{발급받은-API-KEY}",
        "DEFAULT_MINIMUM_TOKENS": "5000"
      }
    },
    "exa": {
      "type": "local",
      "command": ["exa-mcp-server"],
      "environment": {
        "EXA_BASE_URL": "http://{gateway-host}:8000/proxy/exa",
        "EXA_API_KEY": "{발급받은-API-KEY}"
      }
    }
  }
}
```

> `{gateway-host}`와 `{발급받은-API-KEY}`는 IT 담당자로부터 발급받습니다.
> 기존 Nexus MCP 등 다른 설정이 있다면 `"mcp"` 안에 항목만 추가하세요.

## ❓ 자주 묻는 질문

**Q. 검색 결과가 갑자기 이상하게 나와요.**  
A. 검색어에 차단 대상 정보(IP, 계정 정보 등)가 포함되어 치환되었을 가능성이 높습니다. 검색어에서 해당 정보를 제거하고 에러 메시지나 기술 키워드 중심으로 재검색해 보세요.

**Q. 내 검색 기록을 볼 수 있나요?**  
A. 개발자 본인의 검색 기록 열람은 제공되지 않습니다. 보안 목적의 열람은 보안 담당자만 가능합니다.

**Q. 차단 규칙을 직접 수정할 수 있나요?**  
A. 차단 규칙은 보안 담당자만 관리할 수 있습니다. 특정 패턴의 차단 조정이 필요한 경우 보안 담당자에게 문의하세요.

**Q. 인터넷이 안 될 때도 opencode를 쓸 수 있나요?**  
A. Context7/Exa 검색 기능은 MCP Gateway를 통해 인터넷에 연결되므로 Gateway 서버가 정상 운영 중이어야 합니다. Gateway 서버 장애 시 IT 담당자에게 문의하세요.

**Q. 차단이 적용된 사실을 어떻게 알 수 있나요?**  
A. 현재 버전에서는 개발자에게 별도 알림이 발송되지 않습니다. 검색 결과가 기대와 다를 경우 위의 주의사항을 참고해 검색어를 수정해 보세요.
